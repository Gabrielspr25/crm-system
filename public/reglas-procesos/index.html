🔐 Autenticación y Roles

Sistema de login

  • Autenticación con usuario + contraseña (bcrypt + JWT)
  • Token JWT sin expiración fija (persistente)
  • Registro de último login automático en DB
  • Endpoint de refresh token disponible
  • Soporte adicional Microsoft Azure MSAL (SSO corporativo)

Roles del sistema

  • admin — Acceso total, ve todos los datos
  • supervisor — Acceso ampliado a datos del equipo
  • vendedor — Solo ve sus propios datos y ventas
Un vendedor no puede crear ventas para otro vendedor ni ver sus tareas o metas.

Permisos granulares

  • Permisos configurables por usuario independiente del rol
  • Catálogo de permisos definido en permissionCatalog.js
  • Middleware de auth valida token en cada request protegido
  • Token contiene: userId, username, role, salespersonId

Perfiles de acceso (Presets)

  • Presets definidos en tabla permission_presets con efectos JSONB
  • Se asignan a vendedores desde Módulo Vendedores → Editar → Perfil de acceso
  • Al aplicar un preset se escriben user_permission_overrides y se actualiza salespeople.permission_preset_name
  • Vendedores con preset asignado muestran badge morado en su tarjeta
El rol en DB siempre prevalece. El sistema usa /api/me para detectar el rol real (no el token de localStorage).

🔄 Flujo Operativo del Vendedor

1. Búsqueda y análisis del cliente

Localizar al cliente en el sistema para visualizar sus BANs y suscriptores activos. Verificar convergencia (2+ productos).

2. Gestión de servicios

Administrar renovaciones, nuevas ventas, cancelaciones y comparativas de planes. Cotización basada en tarifas Tango en tiempo real.

3. Registrar venta (Deal)

Crear un Deal con tipo de producto y tipo de venta. El sistema genera automáticamente las tareas del workflow correspondiente.

4. Avanzar tareas del workflow

Completar cada paso secuencial del deal. No se puede avanzar al siguiente paso sin terminar el anterior. El sistema activa el próximo paso automáticamente.

5. Seguimiento y follow-up

Gestionar prospectos en seguimiento mediante el módulo de follow-up. Registrar campañas y referidos asociados.

6. Comunicación y cierre

Envío de propuestas y correos desde la plataforma. Export PDF/Excel de reportes para cliente o supervisor.

⚡ Deal Workflow — Reglas de Negocio

Tipos de producto

  • FIJO — Línea fija
  • MOVIL — Línea móvil
  • CLARO_TV — Televisión
  • CLOUD — Servicios cloud
  • MPLS — Red privada

Tipos de venta

  • NEW — Venta nueva
  • REN — Renovación

Toda combinación de producto + tipo de venta genera workflow de tareas automáticamente.

Estados de tarea

  • pending — Esperando turno
  • in_progress — Tarea activa
  • done — Completada
Al completar una tarea (done), el siguiente paso se activa automáticamente a in_progress.

Flujo de progresión de tareas dentro de un deal:

✓ Paso 1 (done)
→
⚡ Paso 2 (in_progress)
→
Paso 3 (pending)
→
Paso N (pending)
Regla estricta: No se puede marcar un paso como done si el paso anterior no está completado. El sistema rechaza la operación.

🔄 Sincronización desde Seguimiento (Pasos tab)

Cómo funciona el Sync

  • En la pestaña Pasos del cliente, el botón Sincronizar tareas lee la matriz de seguimiento activa
  • Mapeo automático de columnas a deals:
    fijo_ren → FIJO/REN, fijo_new → FIJO/NEW,
    movil_nueva → MOVIL/NEW, movil_renovacion → MOVIL/REN,
    claro_tv → CLARO_TV/NEW, cloud → CLOUD/NEW, mpls → MPLS/NEW
  • Solo se sincronizan las combinaciones marcadas como activas en la matriz

Qué hace el endpoint de sync

  • Elimina deals sin tareas completadas (preserva deals con al menos 1 tarea done)
  • Crea nuevos deals por cada combinación activa
  • Genera las tareas del workflow según la plantilla correspondiente
  • Endpoint: POST /api/clients/:clientId/sync
El sync requiere que el cliente tenga un seguimiento activo guardado. Sin matriz de seguimiento, el botón queda deshabilitado.

🎯 Metas y Comisiones

Productos con meta

  • Fijo Renovación
  • Fijo Nueva
  • Móvil Nueva
  • Móvil Renovación
  • Claro TV
  • Cloud

Cálculo de comisiones

  • Metas por vendedor + producto + periodo (año/mes)
  • Comisiones calculadas desde sales_history
  • Retención automática del 10%
  • Neto = Ganado − Retención
  • Porcentaje de cumplimiento = Ganado / Meta × 100

Visibilidad por rol

  • admin / supervisor — Ve resumen global de todos los vendedores + botón "Configurar Metas"
  • vendedor — Solo ve su propia tarjeta de rendimiento
  • Permiso granular goals.configure permite configurar metas sin ser admin
  • Permiso granular reports.admin_view permite ver datos globales
  • El sistema usa /api/me (rol en vivo) en lugar del token de localStorage para evitar permisos obsoletos
El botón "Configurar Metas" y la página /metas/configurar verifican rol en tiempo real. Un cambio de rol en DB tiene efecto sin necesidad de re-login.

📊 Clientes, BAN y Suscriptores

Clientes

  • Convergencia: cliente con 2 o más productos activos
  • Prohibida la creación de clientes duplicados
  • Cada cliente puede tener múltiples BANs

BAN (Business Account Number)

  • Ciclo de vida ligado estrictamente al cliente
  • Cada BAN puede contener múltiples suscriptores
  • Módulo de bans independiente con gestión completa

Suscriptores

  • ID obligatorio de 19 dígitos
  • Contrato siempre debe tener plan y fechas (inicio/fin)
  • Reportes de suscriptores disponibles por periodo

🤖 Motor OCR — Sincronización automática

Al subir capturas de pantalla del portal de Claro, el sistema aplica la siguiente lógica de sincronización sobre los suscriptores:

Estado en sistema Estado detectado en captura Acción aplicada
Activo Activo Actualiza plan del suscriptor
Activo Cancelado Cancela suscriptor en sistema
Activo Suspendido Sin cambios
Suspendido Cualquier estado Sin cambios
El OCR también procesa documentos PDF para extracción de datos. Soporta lectura de facturas y contratos.

🔌 Integraciones Externas

Tango Dealer Management

El CRM se sincroniza con Tango para obtener la base de datos maestra de planes, tarifas y precios en tiempo real. Permite cotización precisa y cálculo automático de comisiones basado en la tabla de márgenes vigente.

Integración IA

Módulo /api/ai disponible para asistencia inteligente dentro del CRM. Utilizado para búsqueda semántica, sugerencias y análisis de datos.

Importador masivo

Importación de datos vía /api/importador. Soporta archivos Excel/CSV. Validación y normalización automática de datos al importar.

POS Integration

Integración con sistema de punto de venta externo vía /api/importador y posIntegration para sincronización de transacciones.

Microsoft Azure MSAL

Login corporativo via Single Sign-On de Microsoft. Permite acceso con credenciales de la organización SS Group.
